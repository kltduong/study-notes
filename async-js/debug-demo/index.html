Async Event Loop — Debug Demo

1. The Initial Example (async stack traces)

Open DevTools → Sources tab. Click the button, then inspect the Call Stack panel when it pauses at the debugger statement inside the setTimeout callback. You'll see the async boundary.

2. Microtask vs Task ordering

Open DevTools → Performance tab. Click "Record", then click the button, then stop recording. You'll see separate task blocks.

3. rAF position in the event loop

Watch the console output order: sync → microtask → rAF → setTimeout (task).

4. Animation: setTimeout vs rAF

Both boxes move 400px. Top uses setInterval(16ms), bottom uses rAF. Watch smoothness difference (especially on high-refresh displays).
